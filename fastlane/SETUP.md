# VoxRider release lanes (local fastlane)

Local, key-file-auth releases for both stores — no CI, no GitHub secrets. Run from the repo root.
(`fastlane/README.md` is auto-generated by fastlane on every run and gitignored — keep notes here.)

## One-time setup
1. `cp fastlane/.env.example fastlane/.env` and fill in the real values (`.env` is gitignored).
2. **iOS** — needs the `Apple Distribution: Naveen Gowda (WKVSGRCJD9)` cert in your login keychain
   and the ASC API key `.p8` at `ASC_KEY_FILEPATH`.
3. **Android** — needs:
   - the upload keystore + `android/keystore.properties` (convention: keep the `.jks` in
     `~/.android-keystores/voxrider-upload.jks`, same as Veloscape), and
   - a Google Play **service-account JSON** at `PLAY_JSON_KEY_FILEPATH`
     (Google Cloud → service account → JSON key; Play Console → Users & permissions → grant app access).

## Usage
```bash
fastlane ios_release          # archive (distribution) → TestFlight (no App Store submit)
fastlane android_release      # signed AAB → Play "internal" track as a DRAFT
scripts/release-android.sh    # bumps versionCode, then runs android_release
```

Both lanes are gated: iOS stops at TestFlight, Android stops at an internal draft, and
`android_release` aborts if the keystore is missing or the AAB comes out debug-signed.

## Bump versions before releasing
- iOS: `CURRENT_PROJECT_VERSION` / `MARKETING_VERSION` in `ios/VoxRider.xcodeproj/project.pbxproj`
- Android: `versionCode` / `versionName` in `android/app/build.gradle` (the script bumps versionCode for you)

Back up all three credentials (keystore + passwords, ASC `.p8`, Play JSON) in your password manager.
